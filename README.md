# 진로 탐색 프로젝트: 연합 학습 모델 기밀성 분석 (DLG Attack Simulation)

이 프로젝트는 **연합 학습(Federated Learning)** 환경에서 발생할 수 있는 데이터 유출 취약점인 **DLG(Deep Leakage from Gradients) 공격**을 시뮬레이션하고 방어/압축 기법이 공격에 미치는 영향을 분석하기 위한 연구용 코드입니다.

주어진 CIFAR-100 데이터셋 이미지에 대해 클라이언트가 계산한 그래디언트를 가로채어, 원래의 훈련 이미지와 라벨을 역으로 복원해내는 과정을 구동해 볼 수 있습니다. 또한 희소화(Sparsification) 및 양자화(Quantization)와 같은 네트워크 그래디언트 압축 기법을 적용하여 이러한 기법들이 DLG 공격을 어떻게 방해할 수 있는지 실험할 수 있습니다.

## 🚀 실행 방법

### 기본 실행 (압축 없음)
별도의 압축 기법을 적용하지 않고 원본 그래디언트를 전송하는 환경에서의 공격 시뮬레이션입니다.
```bash
python main.py
```

### 그래디언트 압축 적용 실행
`--compression` 플래그를 통해 연합 학습에서 통신 비용 감소 및 보안 향상을 위해 사용되는 압축 기법을 적용해 볼 수 있습니다.

1. **희소화 (Sparsification)**: 그래디언트 값 중 절대값이 작은 일정 비율을 0으로 만들어버리는 방식입니다. 기본 삭제 비율은 20%(0.2)입니다.
```bash
python main.py --compression sparsification --sparsity 0.2
```
2. **양자화 (Quantization)**: 그래디언트의 정밀도를 낮추는 기법으로 본 코드에서는 FP32(32비트 부동소수점)를 FP16 기반으로 캐스팅하여 정밀도를 낮춥니다.
```bash
python main.py --compression quantization
```
3. **글로벌 가지치기 (Pruning)**: DLG 논문에서 시도한 데이터 방어 기법으로, 각 레이어 텐서별로 잘라내는 대신 모델 전체의 그래디언트를 기준으로 하위 비율(기본 20%)을 삭제하여 노이즈를 줍니다.
```bash
python main.py --compression pruning --sparsity 0.2
```
4. **Soteria 방어 (Soteria)**: 특정 레이어(주로 마지막 FC 레이어)의 입력 특징(Representation) 중요도를 분석하여, 데이터 유출에 큰 영향을 미치는 기울기만을 선택적으로 0으로 만듭니다.
```bash
python main.py --compression soteria --sparsity 0.2
```

## 📁 주요 코드 설명

이 프로젝트는 기능별로 여러 파이썬 스크립트로 모듈화 되어 있습니다. 각 파일의 상세 역할은 다음과 같습니다.

### 1. `main.py`
**프로젝트의 메인 진입점(Entry Point)**입니다. 
- CIFAR-100 데이터셋을 다운로드하고 여러 클라이언트에게 각기 다른 이미지를 할당합니다.
- `model.py`를 통해 글로벌 AI 모델(ResNet)을 초기화합니다.
- 다수의 클라이언트(`client.py`)와 서버(`server.py`)를 인스턴스화하여, 다중 클라이언트 환경에서 각 클라이언트가 로컬 모델을 학습하고 서버로 그래디언트를 전송하는 연합 학습 과정을 시뮬레이션합니다.
- 연합학습 서버가 전체 그래디언트를 취합하여 평균내기 전, 악의적인 서버가 **특정 클라이언트의 그래디언트를 분리하여 탈취**, `dlg.py`를 호출하여 타겟 클라이언트의 원본 데이터 복원(DLG 공격)을 시도합니다.
- 공격 시뮬레이션 후 정상적인 연합 학습 절차로 돌아와 수집된 전체 클라이언트의 그래디언트를 평균(Aggregation)내어 글로벌 모델을 업데이트합니다.
- 복원된 결과물과 타겟 클라이언트의 원본 이미지를 비교하는 시각화 작업 및 MSE(Mean Squared Error) 평가를 수행합니다.

### 2. `dlg.py`
**향상된 DLG (Deep Leakage from Gradients) 공격 알고리즘**이 구현되어 있습니다.
- 노이즈가 낀 랜덤한 **더미 이미지**와 **더미 라벨**을 생성합니다.
- 더미 데이터를 모델에 통과시켜 나오는 더미 그래디언트와 실제 클라이언트가 전송한 타겟 그래디언트 간의 차이를 최소화하는 방향으로 더미 데이터를 업데이트합니다.
- L2 Distance 대신 방향성을 매칭하는 **코사인 유사도(Cosine Similarity) Loss**를 사용하여 발산을 막습니다.
- 이미지 픽셀 간의 차이를 줄여주는 **TV (Total Variation) Loss** 패널티를 부여하여 복원된 이미지에 생기는 노이즈(아티팩트)를 줄이고 부드럽게 재현합니다.

### 3. `model.py`
**공격 시뮬레이션을 위한 맞춤형 ResNet 모델 구조**가 정의되어 있습니다.
- DLG 공격은 활성화 함수의 미분값이 0이 되어버리면 역추적이 어렵다는 특징이 있습니다. 이를 극복하기 위해 (논문 환경과 동일하게) `ReLU` 대신 **`Sigmoid`** 함수를 사용합니다.
- 모델 층을 지나며 이미지 크기가 줄어들지 않도록 합성곱(Convolution) 시 `stride=1`로 고정하여 입력 이미지(32x32) 해상도가 끝까지 유지되게 합니다.
- 테스트용으로는 가벼운 `ResNet-8` 구조가 기본 적용되어 있으며, 필요 시 원본 두께인 `ResNet-56` 모델을 호출할 수 있습니다.

### 4. `client.py`
**연합 학습의 클라이언트(FLClient)** 역할을 담당합니다.
- 로컬 데이터를 기반으로 Forward/Backward 패스를 수행해 실제 파라미터 그래디언트(Gradient)를 연산합니다.
- 실행 인자에 맞춰 `utils.py`의 `compress_gradients` 함수를 선호출하여 구해진 그래디언트를 압축(희소화/양자화)한 후 서버로 전송할 데이터를 반환합니다.

### 5. `server.py`
**연합 학습의 서버(FLServer)** 역할을 담당합니다.
- 여러 클라이언트들로부터 전송된 (압축된) 그래디언트를 차례로 수신 받아 내부 리스트(`received_gradients_list`)에 일시적으로 저장합니다.
- `aggregate_and_update()`: 수신된 모든 클라이언트의 그래디언트를 평균(Aggregation)내어 서버의 글로벌 모델 파라미터를 업데이트하는 정상적인 연합 학습 과정을 수행합니다.
- `get_target_client_gradients()`: 연합 학습 도중 악의적인 서버가 타겟 클라이언트의 데이터를 탈취할 목적으로, 전체 그래디언트가 평균화되기 전 지정된 클라이언트만의 그래디언트를 단독 추출해내는 악의적 메소드를 포함합니다.

### 6. `utils.py`
**부가 기능 및 시각화 유틸리티 함수**들을 모아둔 파일입니다.
- `get_args()`: 명령줄 인자(CLI arguments)를 파싱합니다.
- `compress_gradients()`: 희소화(Sparsity 값 이하의 값 제거) 및 양자화(float16 변환 후 복귀) 같은 그래디언트 압축 알고리즘이 구현되어 있습니다.
- `visualize_results()`: 공격이 끝난 후 본래의 정답(Ground Truth) 이미지와 공격으로 복원해 낸 이미지를 matplotlib를 이용해 화면에 나란히 띄워줍니다.

## 🕸️ GNN(Graph Neural Network) 모델에서의 DLG 공격 — 멀티시드 엄밀 분석

이미지 실험에 이어 **그래프 신경망(GNN)** 모델 `FedLoGModel`(2-Layer GraphSAGE + 학습 가능한 합성 노드, 참고: Kim et al., *Subgraph Federated Learning for Local Generalization*, ICLR 2025)을 대상으로 DLG 공격을 수행했습니다. WikiCS를 3개 클라이언트 서브그래프로 분할하고, 악의적 서버가 타겟 노드의 그래디언트를 단독 탈취해 노드 특성을 복원합니다.

> ⚠️ **초기 버전 정정**: 초기 README는 이 트랙의 결과를 **시드 고정 없이 단 1회 실행**한 수치(예: 코사인 `0.9999`)로 보고했습니다. 검증을 위해 **원본 코드를 그대로 5회 재실행**하자 코사인 유사도가 **`0.518`~`0.999` (평균 0.85 ± 0.22)** 로 크게 출렁였습니다. 즉 `0.9999`는 운 좋은 단일 실행이었습니다. 아래는 시드를 고정한 **멀티시드(n=5) 엄밀 재실험** 결과입니다. (드라이버: [`experiments/run_gnn.py`](experiments/run_gnn.py))

### 🚨 공격–방어 공방 결과 (미학습 worst-case, n=5)

| 단계 | 시나리오 | 복원 코사인 (mean±std) | 충실도(유용성) | 해석 |
| :--- | :--- | :--- | :--- | :--- |
| ① | Baseline (방어 없음) | `0.384 ± 0.316` | `1.000` | 노드마다 천차만별, 분산이 매우 큼 |
| ② | FedLoG (특징 스케일링) | `0.385 ± 0.447` | `1.000` | 평균상 baseline과 차이 없음 |
| ③ | Adaptive (조인트 최적화) | `0.852 ± 0.174` | `1.000` | 가장 높은 복원 — 공격 우세 |
| ④ | **DP (클리핑+노이즈)** | `0.002 ± 0.013` | `0.001` | **강력·일관된 방어** |
| ⑤ | Sparse (95%) | `0.498 ± 0.320` | `0.702` | 방어되나 DP보다 약함 |

절대값의 분산이 크지만, **각 시드는 같은 타겟 노드를 공유**하므로 시드 내 **쌍체(paired) 비교**를 하면 공방 효과가 매우 일관되게 드러납니다:

| 전환 (시드 내) | 평균 Δ코사인 | 시드별 부호 | 의미 |
| :--- | :--- | :--- | :--- |
| 무방어 → 특징 스케일링 | `+0.001 ± 0.465` | 들쭉날쭉 | 스케일링 단독은 **신뢰할 수 없는 방어** |
| 순진 공격 → 적대적 공격 | `+0.467 ± 0.414` | **전부 양수** | 적대적 공격이 **항상** 스케일링 방어를 무력화 |
| 적대적 공격 → DP | `−0.849 ± 0.168` | **전부 음수** | DP가 **항상·강력하게** 차단 |
| 적대적 공격 → 희소화 | `−0.354 ± 0.223` | **전부 음수** | 희소화도 차단(DP보다 약함) |

**핵심 발견**

1. **단일 실행 보고의 위험성**: GNN 노드 특성 복원도는 타겟 노드/시드에 따라 0~0.99까지 극심하게 변동합니다. 단일 수치(0.9999)는 대표성이 없으며, 반드시 다중 시드 평균±표준편차로 보고해야 합니다.
2. **난독화는 무력하다 (코사인은 스케일 불변)**: 특징 스케일링은 비밀 계수 `s`를 곱해 그래디언트를 가리지만, 복원의 품질 지표인 **코사인 유사도는 스케일에 불변**입니다. 실제로 적대적 공격이 비밀 스케일을 1.81이나 틀리게 추정한 경우(seed 1)에도 특성 *방향*은 코사인 `0.99`로 복원되었습니다. 스케일링이 근본적으로 방어가 못 되는 이유입니다.
3. **정보 훼손형 방어만 견고**: DP(그래디언트 클리핑 + 보정 노이즈)는 적대적 공격조차 일관되게(전 시드) 무력화하여 복원 코사인을 `0.002`로 떨어뜨립니다. 다만 그래디언트 충실도도 `0.001`로 떨어져 **유용성 손실이 큽니다**. 희소화(95%)는 충실도 `0.70`을 유지하며 방어하므로 **프라이버시-유용성 균형이 더 좋습니다**.

### 📉 학습 진행과 유출 (정직한 한계)

공격 전 모델을 FedAvg로 학습시킨 뒤 baseline 공격을 수행한 결과(n=5): 라운드 `0 → 5 → 20`에서 복원 코사인 `0.384 → 0.385 → 0.420`으로 **거의 평탄**했고(정확도 `0.12 → 0.21`로 모델이 아직 충분히 학습되지 않음), 별도 단일 실행(40라운드)에서는 `0.14`로 떨어졌습니다. 즉 **유출은 학습 초기 라운드 내내 높게 유지**되며, 충분한 학습 후에야 감소하는 것으로 보입니다(엄밀한 확인은 향후 과제).
