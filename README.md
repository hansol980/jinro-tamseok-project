# 진로 탐색 프로젝트: 연합 학습 모델 기밀성 분석 (DLG Attack Simulation)

이 프로젝트는 **연합 학습(Federated Learning)** 환경에서 발생할 수 있는 데이터 유출 취약점인 **DLG(Deep Leakage from Gradients) 공격**을 시뮬레이션하고 방어/압축 기법이 공격에 미치는 영향을 분석하기 위한 연구용 코드입니다.

주어진 CIFAR-100 데이터셋 이미지에 대해 클라이언트가 계산한 그래디언트를 가로채어, 원래의 훈련 이미지와 라벨을 역으로 복원해내는 과정을 구동해 볼 수 있습니다. 또한 희소화(Sparsification) 및 양자화(Quantization)와 같은 네트워크 그래디언트 압축 기법을 적용하여 이러한 기법들이 DLG 공격을 어떻게 방해할 수 있는지 실험할 수 있습니다.

---

## 🚀 실행 방법

### 기본 실행 (압축 없음)
별도의 압축 기법을 적용하지 않고 원본 그래디언트를 전송하는 환경에서의 공격 시뮬레이션입니다.
```bash
python main.py
```

### 그래디언트 압축 적용 실행
`--compression` 플래그를 통해 연합 학습에서 통신 비용 감소 및 보안 향상을 위해 사용되는 압축 기법을 적용해 볼 수 있습니다.

1. **희소화 (Sparsification)**: 그래디언트 값 중 절대값이 작은 일정 비율을 0으로 만들어버리는 방식입니다. 기본 삭제 비율은 20%(0.2)입니다.
```bash
python main.py --compression sparsification --sparsity 0.2
```
2. **양자화 (Quantization)**: 그래디언트의 정밀도를 낮추는 기법으로 본 코드에서는 FP32(32비트 부동소수점)를 FP16 기반으로 캐스팅하여 정밀도를 낮춥니다.
```bash
python main.py --compression quantization
```
3. **글로벌 가지치기 (Pruning)**: DLG 논문에서 시도한 데이터 방어 기법으로, 각 레이어 텐서별로 잘라내는 대신 모델 전체의 그래디언트를 기준으로 하위 비율(기본 20%)을 삭제하여 노이즈를 줍니다.
```bash
python main.py --compression pruning --sparsity 0.2
```
4. **Soteria 방어 (Soteria)**: 특정 레이어(주로 마지막 FC 레이어)의 입력 특징(Representation) 중요도를 분석하여, 데이터 유출에 큰 영향을 미치는 기울기만을 선택적으로 0으로 만듭니다.
```bash
python main.py --compression soteria --sparsity 0.2
```

## 📁 주요 코드 설명

이 프로젝트는 기능별로 여러 파이썬 스크립트로 모듈화 되어 있습니다. 각 파일의 상세 역할은 다음과 같습니다.

### 1. `main.py`
**프로젝트의 메인 진입점(Entry Point)**입니다. 
- CIFAR-100 데이터셋을 다운로드하고 여러 클라이언트에게 각기 다른 이미지를 할당합니다.
- `model.py`를 통해 글로벌 AI 모델(ResNet)을 초기화합니다.
- 다수의 클라이언트(`client.py`)와 서버(`server.py`)를 인스턴스화하여, 다중 클라이언트 환경에서 각 클라이언트가 로컬 모델을 학습하고 서버로 그래디언트를 전송하는 연합 학습 과정을 시뮬레이션합니다.
- 연합학습 서버가 전체 그래디언트를 취합하여 평균내기 전, 악의적인 서버가 **특정 클라이언트의 그래디언트를 분리하여 탈취**, `dlg.py`를 호출하여 타겟 클라이언트의 원본 데이터 복원(DLG 공격)을 시도합니다.
- 공격 시뮬레이션 후 정상적인 연합 학습 절차로 돌아와 수집된 전체 클라이언트의 그래디언트를 평균(Aggregation)내어 글로벌 모델을 업데이트합니다.
- 복원된 결과물과 타겟 클라이언트의 원본 이미지를 비교하는 시각화 작업 및 MSE(Mean Squared Error) 평가를 수행합니다.

### 2. `dlg.py`
**향상된 DLG (Deep Leakage from Gradients) 공격 알고리즘**이 구현되어 있습니다.
- 노이즈가 낀 랜덤한 **더미 이미지**와 **더미 라벨**을 생성합니다.
- 더미 데이터를 모델에 통과시켜 나오는 더미 그래디언트와 실제 클라이언트가 전송한 타겟 그래디언트 간의 차이를 최소화하는 방향으로 더미 데이터를 업데이트합니다.
- L2 Distance 대신 방향성을 매칭하는 **코사인 유사도(Cosine Similarity) Loss**를 사용하여 발산을 막습니다.
- 이미지 픽셀 간의 차이를 줄여주는 **TV (Total Variation) Loss** 패널티를 부여하여 복원된 이미지에 생기는 노이즈(아티팩트)를 줄이고 부드럽게 재현합니다.

### 3. `model.py`
**공격 시뮬레이션을 위한 맞춤형 ResNet 모델 구조**가 정의되어 있습니다.
- DLG 공격은 활성화 함수의 미분값이 0이 되어버리면 역추적이 어렵다는 특징이 있습니다. 이를 극복하기 위해 (논문 환경과 동일하게) `ReLU` 대신 **`Sigmoid`** 함수를 사용합니다.
- 모델 층을 지나며 이미지 크기가 줄어들지 않도록 합성곱(Convolution) 시 `stride=1`로 고정하여 입력 이미지(32x32) 해상도가 끝까지 유지되게 합니다.
- 테스트용으로는 가벼운 `ResNet-8` 구조가 기본 적용되어 있으며, 필요 시 원본 두께인 `ResNet-56` 모델을 호출할 수 있습니다.

### 4. `client.py`
**연합 학습의 클라이언트(FLClient)** 역할을 담당합니다.
- 로컬 데이터를 기반으로 Forward/Backward 패스를 수행해 실제 파라미터 그래디언트(Gradient)를 연산합니다.
- 실행 인자에 맞춰 `utils.py`의 `compress_gradients` 함수를 선호출하여 구해진 그래디언트를 압축(희소화/양자화)한 후 서버로 전송할 데이터를 반환합니다.

### 5. `server.py`
**연합 학습의 서버(FLServer)** 역할을 담당합니다.
- 여러 클라이언트들로부터 전송된 (압축된) 그래디언트를 차례로 수신 받아 내부 리스트(`received_gradients_list`)에 일시적으로 저장합니다.
- `aggregate_and_update()`: 수신된 모든 클라이언트의 그래디언트를 평균(Aggregation)내어 서버의 글로벌 모델 파라미터를 업데이트하는 정상적인 연합 학습 과정을 수행합니다.
- `get_target_client_gradients()`: 연합 학습 도중 악의적인 서버가 타겟 클라이언트의 데이터를 탈취할 목적으로, 전체 그래디언트가 평균화되기 전 지정된 클라이언트만의 그래디언트를 단독 추출해내는 악의적 메소드를 포함합니다.

### 6. `utils.py`
**부가 기능 및 시각화 유틸리티 함수**들을 모아둔 파일입니다.
- `get_args()`: 명령줄 인자(CLI arguments)를 파싱합니다.
- `compress_gradients()`: 희소화(Sparsity 값 이하의 값 제거) 및 양자화(float16 변환 후 복귀) 같은 그래디언트 압축 알고리즘이 구현되어 있습니다.
- `visualize_results()`: 공격이 끝난 후 본래의 정답(Ground Truth) 이미지와 공격으로 복원해 낸 이미지를 matplotlib를 이용해 화면에 나란히 띄워줍니다.

## 🕸️ GNN(Graph Neural Network) 모델에서의 DLG 공격 취약성 (FedLoG)

이미지 기반의 CIFAR-100 실험에 이어, 노드(Node) 간의 연결성을 학습하는 **그래프 신경망(GNN)** 모델인 `FedLoGModel`을 대상으로 추가적인 DLG 연합학습 해킹 시뮬레이션을 진행했습니다.

### 📌 실험 개요 및 구현 (`gnn_*.py`)
- **타겟 모델 (`gnn_model.py`)**: 2-Layer GraphSAGE와 학습 가능한 합성 노드를 결합하여 구성된 `FedLoGModel`입니다.
- **참고 논문**: SUBGRAPH FEDERATED LEARNING FOR LOCAL GENERALIZATION
- **공격 알고리즘 (`gnn_dlg.py`)**: 이미지 픽셀 구조에 국한된 TV(Total Variation) Loss를 제외하고, 노드 특성(1D Feature) 벡터를 효과적으로 역산하기 위해 **L2 패널티(정규화)와 코사인 유사도(Cosine Similarity) 매칭 기법**을 적용한 GNN 특화 DLG입니다.
- **시나리오 (`main_gnn.py`)**: WikiCS 데이터셋을 3개의 클라이언트 그래프로 분할하여 연합학습을 수행합니다. 이때 악의적인 서버가 타겟 노드의 그래디언트 정보만 단독으로 가로채어 노드 피처(Features) 복원을 시도합니다.

### 🚨 실험 결과 및 치명적 위험성 (Vulnerabilities)

`main_gnn.py`를 통한 시뮬레이션 결과, 타겟 노드 특성 복원율이 **오차(MSE) `0.000003`**, **코사인 유사도 `0.9999`**를 기록하며 사실상 **100% 완벽하게 원본 데이터가 복원**되었습니다.

1. **완벽한 속성 유출 (Attribute Inference)**
   그래프 데이터는 대개 유저의 소셜 네트워크 프로필, 금융 거래 내역, 진단 기록 등 **고도로 민감한 개인의 특성(Feature)**을 수치화하여 담고 있습니다. 그래디언트만으로 이 수치들이 소수점 단위까지 정확히 역산된다는 것은, 심각한 프라이버시 침해로 이어질 수 있습니다.
2. **구조적 한계 (Lack of Built-in Privacy)**
   `FedLoGModel`과 같은 GNN 기반 연합학습은 클라이언트 측 데이터를 보호하기 위해 고안되었으나, 그 내부에 주고받는 "기울기(Gradient)" 자체에 원본 피처의 단서가 노골적으로 묻어남이 증명되었습니다. 
3. **방어 기법의 부재 시 위험도**
   이미지 데이터보다 차원이 낮거나 단순한 밀집 벡터(Dense Vector) 형태를 띄는 GNN 노드 특성의 경우, 방향성(Cosine Similarity) 위주의 손실 함수 최적화(L-BFGS 등)를 맞으면 방어 체계(차분 프라이버시, Soteria 등)가 없을 때 **오히려 이미지보다 더 빠르고 정확하게 뚫릴 수 있습니다.**

> **🔥 결론**: GNN을 활용한 연합학습(Federated Graph Learning) 시스템을 구축할 때, 단순히 원본 데이터를 전송하지 않는다는 것만으로는 안전을 보장할 수 없습니다. 그래디언트 유출 공격에 대비한 강력한 보호 기법(프라이버시 노이즈 등)이 필수적으로 수반되어야 합니다.

### 🛡️ 논문 기반의 FedLoG 방어 메커니즘 적용 결과
해당 취약점을 극복하기 위해 본 프로젝트에서는 논문(`SUBGRAPH FEDERATED LEARNING FOR LOCAL GENERALIZATION`)에서 제안한 3가지 핵심 방어 기법을 `main_gnn.py`에 적용해 보았습니다. (`--defense` 플래그로 활성화)

1. **데이터 응축 (Data Condensation)**: 원본 데이터가 아닌 합성 노드(`syn_head`, `syn_tail`) 파라미터만 글로벌 데이터로 취합하여 공유.
2. **특징 스케일링 (Feature Scaling)**: 클라이언트가 글로벌 합성 데이터에 자신만의 시크릿 스케일링 계수(Secret Scaling Factor)를 곱해 훈련에 사용하여, 그래디언트를 난독화함.
3. **클래스 분포 노이즈 (Class Distribution Noise)**: 서버 집계 시 사용되는 클래스 비율 분포에 가우시안 노이즈(Gaussian Noise)를 추가하여 데이터의 통계적 분포 유출을 방지.

위 방어 기법들을 켠 상태(`python main_gnn.py --defense`)에서 악의적 서버가 해킹을 시도한 결과는 다음과 같습니다:

- **방어 없음(Baseline)**: MSE `0.000003` / 코사인 유사도 `0.9999` (완벽히 탈취됨)
- **방어 적용(--defense)**: MSE `0.026660` / 코사인 유사도 `0.6782` (형태가 크게 훼손되어 탈취 실패)

방어 체계가 활성화되자 서버는 클라이언트의 비밀 스케일링 팩터를 알지 못해 DLG 공격(L-BFGS 등) 시 최적화 경로를 찾지 못하였고, 결과적으로 **공격에 완벽히 실패**하게 되어 높은 수준의 프라이버시가 확보됨을 확인했습니다.

### ⚔️ 적대적 공격 (Adaptive DLG Attack)을 통한 방어 파훼
공격자(서버)가 해당 방어 기법의 존재를 알아채고, 방어 메커니즘 자체를 역이용하는 정교화된 "적대적 공격(Adaptive Attack)"을 수행하는 시나리오를 추가로 구현했습니다.

공격자는 시크릿 스케일링 팩터를 모른다면, **훔쳐낼 데이터와 함께 '스케일링 팩터'마저도 최적화 대상 변수(Parameter)로 묶어 동시에 추정(Joint Optimization)** 해버리는 전략을 취합니다. (`python main_gnn.py --defense --adaptive_attack`)

- **결과**: 공격자가 최적화를 통해 클라이언트의 시크릿 스케일링 값을 거의 정확히 맞춰냈으며(예: 실제 `2.36` vs 추정 `2.27`), 동시에 원본 데이터까지 **MSE `0.0007`, 코사인 유사도 `0.9984`로 사실상 100% 다시 복원해내는 데 성공**했습니다.

> **💡 중간 결론**: Feature Scaling과 같은 단순한 암호화/난독화 기반 방어는 조인트 최적화(Joint Optimization)를 수행하는 정교한 공격자 앞에서는 결국 파훼될 수 있습니다. 

### 🛡️ 심화 방어 기법 (Advanced Defense Mechanisms) 도입
적대적 공격(Adaptive Attack) 마저 원천적으로 차단하기 위해, 수학적 정보 훼손을 기반으로 하는 두 가지 강력한 심화 방어 기법을 `main_gnn.py`에 추가로 구현했습니다.

1. **차분 프라이버시 (Differential Privacy, DP-SGD)**: `--defense_dp`
   - 클라이언트가 그래디언트를 전송하기 전, 가우시안 노이즈(Gaussian Noise)를 직접 주입합니다.
   - **실험 결과**: 공격자가 스케일과 데이터를 아무리 조인트 최적화하려 해도, 섞여있는 노이즈에 과적합(Overfitting)되어 버립니다. 코사인 유사도가 `0.23`으로 폭락하며 **완벽한 방어(해킹 차단)**에 성공했습니다.

2. **극단적 가지치기 (Extreme Sparsification)**: `--defense_sparse`
   - 그래디언트의 크기(Magnitude) 기준 하위 95%를 0으로 지워버리고(Pruning) 상위 5%의 정보만 서버로 전송합니다.
   - **실험 결과**: 풀어야 할 미지수(데이터 픽셀+스케일 변수)에 비해 제공된 힌트(그래디언트)가 95%나 부족해져 방정식이 성립하지 않습니다. 공격자는 스케일 값을 맞추지 못하고 크게 빗나갔으며, 복원된 데이터 역시 코사인 유사도 `0.82` 수준으로 크게 훼손되어 **방어에 성공**했습니다.

**🔥 최종 결론**: 연합학습 시스템은 방어 기법(스케일링 팩터 등)이 적에게 노출되었을 때조차 안전을 보장할 수 있는 **DP 노이즈 주입**이나 **극단적 정보 압축(Sparsification)** 같은 훼손성 방어 메커니즘이 수반되어야만 완벽한 프라이버시를 달성할 수 있습니다.
